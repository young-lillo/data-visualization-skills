# Document Management Workflow

## Purpose

Keep project docs compact, current, and predictable.

## Responsibilities

- store all project docs under `projects/<slug>/docs/`
- store user assets under `projects/<slug>/docs/assets/`
- prevent ad hoc artifact sprawl
- keep docs aligned with the active project workflow state
- support focused init, update, or summarize passes through `$dv-document-management`
- keep docs logic inside the `dv-document-management` skill with no dedicated prompt file
