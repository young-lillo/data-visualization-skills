# Metabase Core

## Primary Building Blocks

- **Questions**: saved explorations or SQL-backed results
- **Models**: reusable semantic sources for downstream questions
- **Metrics**: trusted business calculations
- **Dashboards**: curated decision surfaces
- **Collections**: content organization and governance boundary
- **Filters**: shared business controls across cards

## Working Order

1. Define the business decisions
2. Identify trusted source tables or models
3. Normalize business metrics and naming
4. Create reusable questions or models
5. Compose dashboards for a clear audience
6. Verify filters, drill paths, and sharing behavior

## Good Metabase Signs

- Stakeholders can read labels without SQL translation
- The same metric means the same thing everywhere
- Collections separate curated assets from exploratory work
- Dashboards feel like guided BI, not unstructured card piles
